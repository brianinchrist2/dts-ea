# Sovereign Bollinger Engine (SBE-2026) - GEMINI Context

## Project Overview
The **Sovereign Bollinger Engine (SBE-2026)** is a sophisticated MQL5 Expert Advisor (EA) designed for the MetaTrader 5 (MT5) platform. It implements a multi-layer Bollinger Bands trading strategy with advanced features like regime detection, signal purification, and risk management.

### Key Features
*   **Regime Oracle:** Dynamically detects market regimes (Strong Trend Up/Down, Mean Reversion, Transition) using FDI and TII indicators.
*   **Signal Purifier:** Filters trading signals through multiple layers (VAF, DABVR, and Order Block Confluence) to reduce false entries.
*   **Execution Engine:** Adapts Bollinger Band parameters based on market volatility (ATR-based).
*   **Risk Sentry:** Implements adaptive stop-loss/take-profit levels and a "MAD-CB" circuit breaker to protect capital.
*   **DTS Integration:** A unique feature that offloads trade execution to an external **DTS service** via the `CDTSClient` module, with a native MT5 fallback mechanism.
*   **Global Gating:** Time and news-based filters to control when the EA is allowed to trade.

## Architecture
The project follows an object-oriented approach in MQL5. The main entry point is `SBE_2026.mq5`, which orchestrates the following modules:
*   `Modules/RegimeOracle.mqh`: Market regime detection.
*   `Modules/SignalPurifier.mqh`: Signal validation and purification.
*   `Modules/ExecutionEngine.mqh`: Strategy logic and adaptive parameters.
*   `Modules/RiskSentry.mqh`: Risk management and trade level calculation.
*   `Modules/GlobalGating.mqh`: Global trading filters.
*   `Modules/DTSClient.mqh`: Integration with the external DTS trading service.
*   `Utilities/`: Support modules for logging, configuration, and indicators.

## Building and Running
### Requirements
*   **MetaTrader 5 (MT5)** Terminal.
*   **MetaEditor 5** (included with MT5) for compilation.
*   **DTS Service:** An external service (default `127.0.0.1:8000`) for trade execution (if enabled).

### Compilation
1.  Open `SBE_2026.mq5` in MetaEditor.
2.  Ensure all dependencies in the `Include/` directory (Modules and Utilities) are present.
3.  Press `F7` to compile. This will generate `SBE_2026.ex5` in the `MQL5/Experts/` folder.

### Execution
1.  Open MT5 Terminal.
2.  Find `SBE_2026` in the **Navigator** window under **Experts**.
3.  Drag and drop it onto a chart.
4.  Configure the **Input Parameters** (DTS host/port, Risk levels, Indicators periods).
5.  Ensure **Allow Algo Trading** is enabled in MT5.

## Development Conventions
*   **Naming Convention:** PascalCase for Classes (e.g., `CRegimeOracle`), camelCase for variables/members (e.g., `lastBarTime`), and UPPER_CASE for constants/macros (e.g., `DTS_API_TIMEOUT`).
*   **Modularity:** Keep logic isolated within specific modules (`.mqh` files).
*   **Logging:** Use the `CLogger` module for consistent information, warning, and error reporting.
*   **Safe Execution:** All trades must pass through the `RiskSentry` and be executed via the `DTSClient` if possible.
*   **Bar-based Processing:** The core logic runs primarily at the beginning of each new bar to avoid over-trading and tick-level noise.
